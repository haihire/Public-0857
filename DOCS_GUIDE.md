## 📊 문서 읽는 순서 추천

```

```
**소요 시간**: 약 20분  
**목표**: 프로젝트가 무엇인지, 어떻게 활용할지 파악

### 중급: 포트폴리오 작성을 시작하는 경우
```
README.md → PORTFOLIO.md → PORTFOLIO_GUIDE_KO.md → ARCHITECTURE.md
```
**소요 시간**: 약 1시간  
**목표**: 전체 내용 이해 및 포트폴리오 작성 시작

모든 문서 정독 → 코드 탐색 → 면접 답변 연습

**소요 시간**: 약 3-4시간  
**목표**: 완벽한 면접 준비 및 프로젝트 완전 이해

## 💡 팁 (Tips)

### 효율적으로 읽기
- 📌 **북마크**: 자주 참고할 섹션은 브라우저 북마크에 추가
- 🔍 **검색**: Ctrl+F (Cmd+F)로 필요한 내용 빠르게 찾기
- 📱 **모바일**: Markdown은 모바일에서도 잘 보입니다

### 포트폴리오 작성 시
- ✍️ **복사하지 말고**: 이해한 후 자신의 말로 다시 작성
- 🎨 **커스터마이즈**: 자신의 경험과 스타일에 맞게 조정
- 📊 **데이터 활용**: 구체적인 수치와 결과 강조

### 면접 준비 시
- 🗣️ **말로 연습**: 문서를 읽은 후 소리 내어 설명 연습
- ❓ **질문 예상**: 각 챌린지에 대한 추가 질문 대비
- 💻 **코드 준비**: 주요 코드는 바로 설명할 수 있도록
